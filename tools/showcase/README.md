# Showcase pipeline

The showcase species in `samples/documents/*.canopyproj` are generated by
script, exported through `canopy-cli`, and rendered into the reference images
in `samples/renders/`. The images are checked in as visual baselines: when a
change alters observable geometry, regenerate them in the same commit and
call the change out in the PR (golden-update rule in
`24_AGENT_IMPLEMENTATION_PLAYBOOK.md`).

## Regenerate everything

From the repo root:

```bash
python3 tools/showcase/generate_documents.py       # writes samples/documents/
cmake --build --preset headless-dev
for t in samples/documents/*.canopyproj; do
  build/headless-dev/apps/cli/canopy-cli export "$t" \
    --preset tests/presets/obj-diagnostic.json \
    --out "build/showcase/$(basename "$t" .canopyproj)"
done
python3 tools/showcase/render_showcase.py          # writes samples/renders/
```

Additional tools: `render_animations.py` (wind/growth/season GIFs),
`render_lod.py` (profile comparison strip), `render_forest.py` (compiles
species to `.canopyrt` and bakes a runtime CPU forest via
`canopy-runtime-demo`). All need `numpy` and `Pillow`.

`render_showcase.py` needs `numpy` and `Pillow`. It is a diagnostic
flat-shaded rasterizer over the exported OBJ files (colors come from the
exported `.mtl`); the WebGPU viewport replaces it eventually (ADR-0005).

## Adding a species

1. Add the document to `generate_documents.py` (materials, generators, seed).
2. Add a solo `render(...)` call — and a slot in the grove scene — in
   `render_showcase.py`.
3. Run the pipeline above and commit the new `samples/documents/…` and
   `samples/renders/…` together with the scripts.
